# extraction-framework · 提取方法论

distill-forge 的"怎么从素材里抽出认知操作系统"的核心方法论。被 SKILL.md 的
**Phase 0B**(need→framework 反推)、**Phase 2**(三重验证 + 五层提取)直接引用。

本文件只讲**方法**(怎么判、怎么抽、按 `fame_tier` 怎么收放),不讲流程闸门(那在 SKILL.md)。
所有判定结果最终落到 `references/research/quotes.jsonl`、`references/research/exclusivity-baseline.md`、
`references/eval/eval.json`,字段遵守 SKILL.md 末尾的「约定 / CONTRACT」。

> **最高原则(继承自 SKILL.md):标注了局限的诚实 60 分,胜过看起来 90 分的伪造品。**
> 这套方法论的全部目的,是把"AI 凭训练记忆造人格"挡在 Phase 2 之外。

---

## 1. 三重验证 —— 一条候选洞见能否升级为「心智模型」

对 Phase 1 采集到的每一条候选洞见(从 `quotes.jsonl` 聚类而来),依次过三关。
三关全过 = **心智模型(mental model)**;过 1–2 关 = 降级为**决策启发(heuristic)**;一关不过 = **丢弃**。
最终保留的心智模型数量,**上限按 `fame_tier`**(见第 4 节的密度自适应表)。

### 关 1 · 复现性(Reproducibility)
同一个认知模式,是否在**多个独立场景**里重复出现?不是一次妙语,而是稳定习惯。

| `target_type` | 复现性闸门 |
|---|---|
| `person` | 该模式须在**≥2 个不同领域/话题**中被本人独立使用(如芒格的"逆向思考"既用于投资也用于人生决策) |
| `methodology` | 该原则须在**≥2 个不同案例/项目**中被验证有效,且不是同一案例的换皮复述 |

操作:在 `quotes.jsonl` 里给候选模式打标聚类,数它命中的**不同 `model` 来源 + 不同话题**。
单一来源反复说同一句 ≠ 复现性(那只是口头禅,归到 Expression DNA 层,不是心智模型)。

### 关 2 · 生成力(Generativity)
这条洞见能不能**推出对一个新问题的非平凡判断**?
"非平凡" = 不是常识、不是套话,换个普通人答不出或答得更泛。

操作:挑 1 个素材里**没出现过**的问题,用这条洞见推演一个具体结论。
若推出来的结论"谁都会这么说"(如"要重视用户"),则生成力不足 → 顶多算决策启发。
若能推出一个具体、可能让人意外、且与这条洞见逻辑自洽的判断 → 过。

### 关 3 · 排他性(Exclusivity)—— 对比式盲测
这条洞见是**这个人特有的**,还是任何聪明通才都会给的答案?用盲测裁决,**不靠主观感觉**。

盲测协议(结果落盘到 `references/research/exclusivity-baseline.md`):
1. 取一组测试题(建议 5–8 题,覆盖该人的核心 `focus` 维度)。
2. **spawn 一个盲 agent**:只给它**问题本身**,**不给候选心智模型、不给人名、不给任何素材**,让它产"通才答案"。
3. 把候选模型驱动出的答案,与盲基线逐题对比。**判过的条件(全部满足)**:
   - 在**多数测试题**上,候选答案比盲基线**更具体且方向不同**(不是同一答案的措辞变体);
   - 且能为"分歧"写出**一行明确说明**:`分歧在哪 + 为何这个人会这样想`。
4. **说不出具体分歧 → 排他性不过 → 降级**(说明这条洞见其实是通才常识,不该归到这个人头上)。

> 排他性是反幻觉的最后一道闸:它专门抓"AI 把通用聪明话冒充成某人独有视角"。

### 判定总表
| 通过关数 | 结论 | 去向 |
|---|---|---|
| 3/3 | **心智模型** | 进 SKILL.md 主体,数量受 `fame_tier` 上限约束 |
| 1–2 | 降级为**决策启发** | 进"决策启发"层,措辞更弱、标依据 |
| 0 | **丢弃** | 不进 skill;若曾被声称一手,在 `weakness_record` 留痕 |

## 2. 五层提取 —— 从素材里抽出来的五种东西

一个目标 skill 的认知操作系统由这五层构成。**由表及里、由硬到软**:越靠下越主观,
越要靠一手佐证 + 三重验证撑住。每层都说明"怎么抽"。

### Layer 1 · Expression DNA(表达指纹)
**抽什么**:这个人怎么说话的**可测量特征**——句式、口头禅、节奏、修辞偏好。
**怎么抽**:
- 喂 `01-著作 / 02-对话 / 03-表达` 的清洗文本给 `scripts/expression_dna.py`,得 6 个句法指标
  (`avg_sentence_len / interrogative_ratio / analogy_per_1k / first_person_ratio / certainty_ratio / transition_per_1k`)。
- 在多份样本上算,取**容差带**(min/max 区间),写进 `eval.json` 的 `dna_targets`。
- 再人工补 7 条风格轴(如 具体↔抽象、激进↔克制、第一性↔类比驱动 等)定性描述。
- 口头禅/标志性比喻直接从 `quotes.jsonl` 摘原句,标 `tier`。
**注意**:DNA 是**软信号**,Phase 4 里单短样本漂移不算失误(见 SKILL.md Phase 4)。

### Layer 2 · 心智模型(Mental Models)
**抽什么**:此人**反复用来切问题的底层框架**(过了三重验证 3/3 的那些)。
**怎么抽**:对每个心智模型写四件套——
1. **一句话定义**(它是什么);
2. **触发条件**(遇到什么类型的问题会启用它)——这条直接喂给 `research-protocol.md`;
3. **一手锚点**(至少一条 `tier:"一手"` 的 `quotes.jsonl` 引用 `id`);
4. **一个工作示例**(用它推一个素材外问题,即关 2 的产物)。

### Layer 3 · 决策启发(Decision Heuristics)
**抽什么**:可操作的经验法则(if–then 级),含**只过 1–2 关被降级**的洞见。
**怎么抽**:统一写成 `当 <情境> 时,倾向于 <动作/取舍>,因为 <依据>`。
- `<依据>` 必须可追到 `quotes.jsonl`(一手最佳;二手须标 `tier`);无依据的启发不要写。
- 数量建议 5–10 条;`fame_tier=low` 时宁缺毋滥,措辞加"基于有限素材"。

### Layer 4 · 反模式 / 价值边界(Anti-patterns & Value Boundaries)
**抽什么**:此人**明确反对/拒绝做**的事,以及他价值观里的**内部张力**(别抹平成完美人设)。
**怎么抽**:
- 从 `05-决策.md` 抓"放弃过/拒绝过什么"(决策的**负空间**往往比正面主张更有判别力)。
- 保留矛盾:若素材显示他在 A 场合重 X、B 场合重相反的 Y,**两条都写**并标情境,不要二选一伪造一致性。
- 写成 `不会做 X / 警惕 Y`,各带一手锚点。这层是 roleplay 模式防 OOC、advisor 模式标"此问题超出其价值边界"的依据。

### Layer 5 · 诚实局限(Honest Limitations)
**抽什么**:这个 skill **不该被信任**的地方——素材盲区、时间过期、领域外、置信度低的维度。
**怎么抽**:
- 列出**素材里他从未公开谈过**的话题 → 对应问题应触发对冲(写进 `eval.json` 的 `edge_case_hedge`)。
- 标注每层的**置信度**(见第 4 节置信度分级),低源维度显式写"置信度=低,基于 N 条素材 / X 条一手"。
- 记录 `fame_tier`、一手占比、是否单模型验证等元信息,汇入 SKILL.md footer 与 `weakness_record`。
- **这一层永远不能为空**。空的诚实局限 = 没做诚实局限。

## 3. Phase 0B · need → framework 反推表

当用户给的是**模糊需求**(如"我想提升投资决策"),用此表把需求反推成"该蒸馏谁/什么视角"。
用法:匹配需求行 → 给 ≤3 个差异化候选(本地已装的 `*-perspective` skill 优先)→ 每个标核心镜头/为何适合/局限,**不替用户硬选**。
"推荐类型"用泛化的**角色原型**,避免在方法论文件里钉死具体人名;实例化时再落到具体候选。

| 用户需求 | 推荐蒸馏的人 / 视角类型 | 核心镜头 | 为何适合 |
|---|---|---|---|
| 投资 / 资本配置 | 价值投资者 / 逆向思考者原型 | 安全边际、误判心理学、机会成本 | 决策可证伪、长期反馈闭环清晰,洞见复现性强 |
| 产品 / 体验取舍 | 极简产品操盘手原型 | 第一性原理、做减法、为谁不做 | 取舍即判别力,反模式(拒绝做什么)信息量大 |
| 写作 / 表达 | 非虚构作者 / 专栏作家原型 | 结构先行、具体压抽象、删形容词 | Expression DNA 可直接测量,样本充足 |
| 谈判 / 冲突处理 | 危机谈判 / 博弈实践者原型 | 战术同理、标签化、给对方台阶 | 启发高度 if–then 化,易转可操作步骤 |
| 团队 / 管理 | 工程文化型管理者原型 | 高标准、直接反馈、所有权 | 价值边界与反模式丰富,内部张力真实 |
| 学习 / 知识体系 | 多学科自学者 / 费曼式讲解者原型 | 多元思维格栅、由教促学、第一性拆解 | 生成力强:能推到素材外的新领域 |
| 创业 / 0→1 | 连续创业者原型 | 反共识下注、速度优先、止损纪律 | 决策负空间(放弃过什么)记录密集 |
| 演讲 / 公开表达 | 故事型演讲者原型 | 叙事弧线、停顿与节奏、单一核心信息 | DNA 与节奏指标可量化,排他性易测 |
| 决策 / 判断质量 | 概率思维 / 决策科学实践者原型 | 期望值、贝叶斯更新、对冲不确定 | `edge_case_hedge` 天然契合,诚实局限好写 |
| 抗压 / 情绪与韧性 | 斯多葛式实践者 / 高压职业者原型 | 控制二分、最坏情形预演、自我对话 | 启发可操作、跨情境复现,价值边界清晰 |

落地提示:给候选时各附一行**局限**(如"逝者→无时间敏感更新"、"小众高手→`fame_tier=low`,置信度低")。

## 4. fame_tier 数据密度自适应 + 置信度分级

### 4.1 密度自适应规则(与 SKILL.md Phase 1 表一致,在此细化"额外动作")
`fame_tier` 决定**心智模型上限**(第 1 节判定的 3/3 候选最多保留几个)、**一手占比闸门**、以及额外验证动作。

| `fame_tier` | 心智模型上限 | 一手占比闸门 | 额外动作 |
|---|---|---|---|
| `high`(全球级名人,预训练海量覆盖) | **5–7** | **> 70%** | Phase 4 **强制冷召回污染测试**:无工具 agent 凭记忆能复现的引用/立场,须有一手佐证,否则降级;整套人格无语料即可重建 = 验证失败,返工 |
| `mid`(有少量访谈/播客) | **3–5** | **> 50%** | 标准流程;边缘心智模型优先砍,保高置信度的 |
| `low`(资料稀疏的小众高手) | **2–3** | 尽力,**无硬闸** | 全局措辞更保守;`honest boundaries` 写明"基于 N 条素材 / X 条一手,置信度=低";超过上限的候选一律降级为决策启发或丢弃 |

闸门未达标时的处置:
- `high/mid` 一手占比未过闸 → **不得进 Phase 2 合成**,回 Phase 1 补采或下调 `fame_tier`(并据实重设上限)。
- 任何 tier,某层全是二手/推测 → 该层整体标"置信度=低",advisor 模式须在输出里显式声明。

### 4.2 置信度分级(低源时如何诚实标注)
每条洞见 / 每层 / 整个 skill 都带一个置信度,锚定在**证据强度**而非语气。

| 置信度 | 触发条件(满足其一即落档) | 标注方式 |
|---|---|---|
| **高** | ≥2 条一手锚点 + 通过三重验证 3/3 | 正常陈述,可直接驱动 roleplay/advisor |
| **中** | 1 条一手 + 部分二手佐证,或 2/3 关通过 | 降级为决策启发;措辞加"倾向于/通常",标来源 tier |
| **低** | 仅二手/推测,或单源,或 `fame_tier=low` 全局 | 必须前置 caveat:"以下基于有限/二手素材,置信度低,可能含推测";写入 `weakness_record` |
| **不可验证** | 声称一手但 `span` 核验失败(`quality_check.py` 已自动降级为 `推测`) | 不得作为心智模型依据;在 `weakness_record` 记 `ungrounded/cannot-verify` |

诚实标注的硬规则:
- 不准用语气掩盖弱证据。低置信度的判断**不允许**写成确定句。
- 单模型验证的 skill 一律标 `single-model, unverified for prior bias`(对应 SKILL.md Phase 4),不谎称独立。
- `fame_tier=low` 时,即便某条侥幸过了三重验证,整体仍默认"中"封顶,除非有 ≥2 条一手把它顶到"高"。
- 所有这些标注最终收敛到:`eval.json`(`first_hand_ratio_baseline` / `weakness_record` / `cold_recall_contaminated`)+ 目标 SKILL.md 的 footer。
